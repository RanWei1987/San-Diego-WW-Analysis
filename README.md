## Introduction

San-Diego-WW-Analysis is a module used by the San Diego Public Health Laboratory for post-analysis processing of wastewater sequencing data.

In a typical wastewater sequencing workflow, raw sequencing data (FASTQ files) generated by an Illumina sequencer are analyzed using the [Freyja](https://github.com/andersen-lab/Freyja) package developed by Kristian Andersen's lab at UC San Diego. This analysis generates the relative abundance (ratio distribution) of SARS-CoV-2 (SC2) lineages detected in wastewater samples. The Freyja workflow is executed on the [Terra](https://terra.bio) platform.

The San-Diego-WW-Analysis module is then used to download batch run data from Terra and further aggregate lineage distributions into 20 major lineage categories.

This module is built upon the following packages:

1. The [SARS-CoV-2_WasteWater_San-Diego](https://github.com/andersen-lab/SARS-CoV-2_WasteWater_San-Diego) package from Kristian Andersen's lab at UC San Diego.  
   This package performs lineage collapsing into major types and was used in the [wastewater surveillance collaboration](https://searchcovid.info/dashboards/wastewater-surveillance/) between the San Diego County Public Health Laboratory and Andersen Lab.  
   The San-Diego-WW-Analysis module serves as a follow-up and extension of this work.

2. The [terra-tools (terratools)](https://github.com/broadinstitute/terra-tools) package from the Broad Institute, which is used for downloading sequencing data from Terra.

## How to Run Locally

### Prerequisites

- Install the Google Cloud SDK: https://cloud.google.com/sdk/downloads  
- Set up Application Default Credentials `gcloud auth application-default login`  
### Setup and Execution
1. Download or clone the repository to your local machine.
2. Create a Python virtual environment (Python 3.9 or 3.10 recommended, use Python3.10 as example):
```
python3.10 -m venv wastewater
```
Note: Conda is not recommended, as it may not provide all required dependencies.

3. Install dependencies:
```
pip install -r requirements-py310.txt
```
4. Run the pipeline:
```
sd_ww_processing-main\SanDiegoWW.bat TERRA_WORKSPACE TERRA_TABLE RUN_FOLDER DUMP_FOLDER
```
`TERRA_WORKSPACE`: The target Terra workspace where the Freyja analysis run was completed  
`TERRA_TABLE`: The data table name within the Terra workspace  
`RUN_FOLDER`: Path to the sd_ww_processing-main directory containing all scripts  
`DUMP_FOLDER`: Directory where downloaded intermediate files will be stored. This directory will be automatically created if it does not exit  
Note: if the execution fails due to google cloud crash during SSL certificate verification, you can try to disable the verification procedure: `gcloud config set auth/disable_ssl_validation  True`  
### Processing and Output
The module performs the following operations:  
1. Downloads and extracts Freyja analysis results for each sequencing sample  
2. Filters out samples with less than 60% genome coverage  
3. For samples passing the quality threshold, extracts lineage distribution data  
4. Collapses lineages into major categories  

Finally, the results are appended to the final output file: `RUN_FOLDER\SD_Sewage_Seqs_Output.csv`
